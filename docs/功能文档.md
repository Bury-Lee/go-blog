# GoBlog 功能文档

> 本文档基于当前仓库代码实现整理，重点介绍项目已经具备的博客能力、社区能力和系统特点。部分代码中存在 `TODO` 的能力会单独标注为“预留”或“待完善”，避免与已实现功能混淆。

## 1. 项目定位

GoBlog 是一个面向博客与轻社区场景的后端项目，使用 `Gin + GORM + Redis + Elasticsearch` 构建，围绕“内容发布、互动沉淀、社交连接、站点运营”四条主线组织功能。

从代码实现来看，它不只是一个简单的文章接口集合，而是把以下几类能力放在了同一个系统里：

- 博客内容管理：文章、分类、搜索、评论、收藏、浏览记录
- 社区互动能力：点赞、消息通知、关注、粉丝、好友聊天
- 运营配置能力：站点信息、SEO、轮播图、友情链接、推荐位
- 平台基础能力：JWT 鉴权、验证码、邮箱验证、日志、缓存同步、AI 能力接入

## 2. 核心特点

### 2.1 博客与社区一体化

项目既支持传统博客站点常见的文章发布、分类、搜索、评论、收藏，也支持关注、私聊、消息中心等偏社区化功能，适合做个人博客升级版、内容社区或者技术分享站点。

### 2.2 内容体系完整

围绕一篇文章，系统已经形成比较完整的生命周期：

- 创建文章
- 文章草稿 / 待审核 / 已发布状态流转
- 文章搜索与详情查看
- 浏览记录沉淀
- 点赞、评论、收藏等互动
- 消息通知回流给作者

### 2.3 Elasticsearch 驱动全文搜索

文章搜索依赖 `Elasticsearch`，支持：

- 标题、摘要、正文联合搜索
- 标签过滤
- 高亮返回
- 按最新、评论数、点赞数、收藏数等维度排序
- “猜你喜欢”式的兴趣搜索扩展

这使项目在搜索体验上明显优于只靠数据库模糊查询的实现方式。

### 2.4 Redis 承担高频计数与限流

项目把浏览量、点赞数、收藏数、评论数等高频统计先写入 Redis，再由定时任务批量同步到数据库，兼顾了接口响应速度与最终一致性。

同时 Redis 还被用于：

- 验证码和邮箱验证码存储
- JWT 黑名单控制
- 防攻击限流
- 浏览去重

### 2.5 AI 能力已接入业务流程

当配置开启 AI 后，项目不只是提供一个通用聊天接口，还把 AI 用在了业务场景里：

- 文章内容审核
- 评论内容审核
- 注册昵称审核
- 站点 AI 助手对话

这让系统具备了基础的智能辅助能力。

### 2.6 兼顾运营与后台管理

除内容本身外，系统已经具备站点级运营配置能力，包括：

- 站点信息配置
- SEO 信息配置
- 首页轮播图管理
- 友情链接管理
- 友站推荐 / 友链推广位管理
- 日志查看与删除

## 3. 功能清单

### 3.1 用户与认证

已实现能力：

- 邮箱验证码发送
- 邮箱注册
- 用户名密码登录
- 邮箱登录
- Access Token / Refresh Token 双令牌机制
- 获取当前用户详情
- 查看指定用户基础信息
- 更新个人资料
- 管理员更新用户资料

- QQ 登录配置已预留，但代码中明确标注为未完成
- 更细粒度的登录 / 密码安全策略仍可继续增强

### 3.2 文章系统

已实现能力：

- 创建文章
- 编辑文章
- 删除文章
- 管理员删除文章
- 文章列表查询
- 文章详情查看
- 文章分类创建 / 查询 / 删除
- 文章全文搜索
- 文章置顶与取消置顶
- 管理员审核文章
- 文章点赞 / 取消点赞
- 文章浏览记录创建、查询、删除
- 收藏文章到收藏夹
- 收藏夹创建、修改、删除

功能说明：

- 文章支持 `草稿`、`待审核`、`已发布` 等状态

文章体验亮点：

- 搜索结果支持高亮
- 支持按评论数、点赞数、收藏数等排序
- 浏览量、点赞量、收藏量、评论量使用缓存增量同步

### 3.3 评论系统

## 4. 架构与实现特点

### 4.1 技术栈

- Web 框架：`Gin`
- ORM：`GORM`
- 缓存：`Redis`
- 搜索：`Elasticsearch 7`
- AI 接入：`go-openai`
- 定时任务：`robfig/cron`

### 4.2 数据层特点

- 支持数据库读写分离
- 第一个数据库配置作为写库，后续配置可作为读库
- 高频计数通过 Redis 缓冲，再批量回写数据库

### 4.3 接口组织特点

- 统一接口前缀为 `/api`
- 静态资源通过 `/web` 暴露
- 路由按用户、文章、评论、消息、聊天、站点等模块拆分

### 4.4 安全相关能力

- JWT 鉴权
- 管理员权限校验
- 验证码校验
- 邮箱验证码校验
- Redis 黑名单控制
- IP 级请求频率限制
- XSS 过滤

## 5. 典型业务流程

### 5.1 内容发布流程

1. 用户登录后创建文章
2. 系统校验分类、内容和权限
3. 如开启 AI / 审核策略，文章进入审核判断
4. 文章以草稿、待审核或已发布状态入库
5. 发布后的文章可被搜索、浏览、点赞、评论、收藏

### 5.2 评论互动流程

1. 用户对已发布文章发表评论
2. 系统建立评论树关系
3. 如开启 AI，评论先经过审核
4. 系统向文章作者、父评论作者或根评论作者发送通知
5. 评论计数通过 Redis 增量记录并异步同步

### 5.3 搜索流程

1. 用户输入关键词或标签
2. 后端使用 ES 对标题、摘要、正文构建查询
3. 搜索结果返回高亮信息
4. 再按文章 ID 回查数据库，补足作者、分类等完整数据

## 6. 适用场景

这个项目比较适合以下类型的站点：

- 个人博客后端
- 技术社区后端
- 轻量内容平台
- 带运营位和消息中心的知识分享站点
- 需要本地 AI 审核 / 助手能力的博客系统

## 7. 当前边界与说明

为避免误解，以下内容需要特别说明：

- QQ 登录处于预留状态，不应在对外文档中写成已正式可用
- 一些聊天与消息细节仍有待完善，例如拉黑、撤回、部分细粒度通知
- 文档中的“已实现”以当前代码路由与主要处理逻辑为准，不代表所有边缘场景都已经完全打磨
- 仓库中的 `otherSoftware` 是额外的数据同步方案，不是博客主功能必需模块

## 8. 总结

从当前实现来看，GoBlog 的最大特点不是“能发文章”，而是已经把文章发布、搜索、互动、社交、运营配置和 AI 接入整合在同一套后端里。

如果配合一个完整前端，它可以较快落地为一个具备以下能力的博客平台：

- 能发布和管理内容
- 能沉淀用户互动
- 能支持简单社区关系
- 能进行站点运营配置
- 能接入 AI 做审核与助手

这也是该项目相比普通博客后端更有特色的地方。
