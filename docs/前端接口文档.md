# GoBlog 前端接口文档（按当前后端代码整理）

## 1. 说明
- 文档来源：直接扫描 `router`、`api`、`models` 当前实现代码。
- 接口前缀：`/api`
- 静态资源前缀：`/web`
- 统一响应结构：
```json
{
  "code": 200,
  "data": {},
  "message": "成功"
}
```
- 统一分页参数（多数列表接口复用）：
  - `page`：页码，默认 `1`，有效范围 `1~20`
  - `limit`：每页数量，默认 `10`，有效范围 `1~40`
  - `key`：模糊搜索关键字（按接口定义）
  - `order`：排序字段（按接口定义）

## 2. 鉴权与请求约定
- AccessToken 传法（绝大多数鉴权接口）：
  - 请求头：`token: <AccessToken>`
  - 或查询参数：`?token=<AccessToken>`
- 刷新 AccessToken 接口例外：
  - 请求头：`Authorization: Bearer <RefreshToken>`
- 图形验证码中间件接口（如发送邮箱验证码、登录）要求请求体包含：
  - `captchaID`
  - `captchaCode`
- 批量删除通用请求体（多个接口复用）：
```json
{
  "IDList": [1, 2, 3]
}
```

## 3. 枚举定义（前端建议直接按数值处理）
- 用户角色 `role`：
  - `1` 管理员
  - `2` 超级会员
  - `3` 会员
  - `4` 普通用户
  - `5` 访客
  - `6` 封禁用户
- 文章状态 `status`：
  - `0` 草稿
  - `1` 审核中
  - `2` 已发布
  - `3` 已下线
- 消息类型 `type`：
  - `0` 评论通知
  - `1` 回复通知
  - `2` 点赞通知
  - `3` 收藏通知
  - `4` 私信
  - `5` 系统通知
  - `6` @我
- 聊天消息类型 `msgType`（位运算）：
  - `1` 文本
  - `2` 图片
  - `4` Markdown

## 4. 通用对象字段（高频）
- `Model` 基础字段：
  - `id`
  - `createdAt`
  - `updatedAt`
- `UserModel` 常见返回字段：
  - `id, username, nickname, avatar, abstract, registerSource, Age, contactInfo, email, openID, role, likeTags, lastLoginTime, IP, createdAt, updatedAt`
- `ArticleModel` 常见返回字段：
  - `id, title, abstract, content, categoryID, tagList, cover, userID, lookCount, diggCount, commentCount, collectCount, openComment, status, createdAt, updatedAt`

## 5. 站点配置模块

### 5.1 获取 QQ 登录地址
- `GET /api/site/qq_login`
- 鉴权：否
- 返回：QQ OAuth URL 字符串

### 5.2 读取站点配置
- `GET /api/site/:name`
- 鉴权：
  - `name=site` 时无需管理员权限
  - 其他 `name` 需要管理员
- `name` 可选值：
  - `site`：公开站点配置
  - `email`：邮箱配置（敏感字段脱敏）
  - `qq`：QQ 配置（敏感字段脱敏）
  - `ai`：AI 配置（敏感字段脱敏）

### 5.3 更新站点配置（管理员）
- `PUT /api/site/:name`
- 鉴权：管理员
- 路径参数 `name` 可选：`site | email | qq | qiNiu | ai`
- 请求体：对应配置对象（完整 JSON）
- 返回：更新结果消息


### 7.4 获取图片文件
- `GET /api/image?id=<imageID>`
- 鉴权：否
- 返回：文件流（不是 JSON）

## 10. 用户模块

### 10.1 发送邮箱验证码
- `POST /api/user/send_email`
- 中间件：图形验证码校验 + 业务类型一致性校验
- Body：
```json
{
  "type": "注册/重置密码/重置邮箱",
  "email": "xx@xx.com",
  "captchaID": "xxx",
  "captchaCode": "1234"
}
```
- 返回：
  - 普通场景：`{ "emailID": "xxx" }`
  - 重置邮箱：`{ "emailID": "新邮箱ID", "resetEmailID": "原邮箱ID" }`

### 10.2 邮箱注册
- `POST /api/user/email`
- 中间件：邮箱验证码校验
- Body：
```json
{
  "EmailID": "发送接口返回的emailID",
  "EmailCode": "邮箱验证码",
  "password": "123456",
  "nickName": "昵称"
}
```
- 返回：
```json
{
  "AccessToken": "...",
  "RefreshToken": "..."
}
```

### 10.3 登录
- `POST /api/user/login`
- 中间件：图形验证码校验
- Body：
```json
{
  "type": "邮箱/用户名",
  "val": "邮箱地址或用户名",
  "pwd": "密码",
  "captchaID": "xxx",
  "captchaCode": "1234"
}
```
- 返回：`AccessToken + RefreshToken`

### 10.4 获取当前登录用户详情
- `GET /api/user/detail`
- 鉴权：是
- 返回：`UserDetailResponse`（基础信息 + 用户配置字段）

### 10.5 获取指定用户基础信息
- `GET /api/user/info/:id`
- 鉴权：否
- 返回：`UserBaseInfoResponse`
  - `userID, age, nickName, avatar, lastLoginTime, region, existDay, articleCount, fansCount, followCount`

### 10.6 登录日志列表
- `GET /api/user/loginlog`
- 鉴权：是
- Query：
  - `type` 必填：`user | admin`
  - `userId, ip, addr, startTime, endTime, page, limit, key, order`
- 返回：`list + count`

### 10.7 重置邮箱
- `PUT /api/user/resetEmail`
- 鉴权：是
- 中间件：先校验原邮箱验证码，再校验新邮箱验证码
- Body 至少需要：
```json
{
  "ResetEmailID": "原邮箱ID",
  "ResetEmailCode": "原邮箱验证码",
  "emailID": "新邮箱ID",
  "emailCode": "新邮箱验证码"
}
```

### 10.8 更新个人资料
- `PUT /api/user/update`
- 鉴权：是
- Body（全字段可选，按需提交）：
```json
{
  "avatar": "...",
  "abstract": "...",
  "likeTags": ["Go", "后端"],
  "nickName": "...",
  "Age": 18,
  "contactInfo": {"github":"..."},
  "openCollect": true,
  "openFollow": true,
  "openFans": true,
  "homeStyleID": 1
}
```

### 10.9 管理员更新用户
- `PUT /api/user/admin/update`
- 鉴权：管理员
- Body：
```json
{
  "userID": 1,
  "username": "new_name",
  "nickname": "new_nick",
  "avatar": "https://...",
  "abstract": "...",
  "role": 4
}
```

### 10.10 刷新 AccessToken
- `POST /api/user/token`
- 请求头：`Authorization: Bearer <RefreshToken>`
- 返回：新的 `AccessToken`（字符串）

## 16. 前端实现注意事项（必须看）

- `GET /api/msg` 当前服务端使用 `ShouldBindJSON`，建议按“GET + JSON Body”发送，或改后端再上。
- `PUT /api/banner/:id` 当前实现中存在绑定对象错误风险，可能导致更新不生效；前端若遇到该问题需后端修复。
- `DELETE /api/article/admingTop` 路径拼写是 `admingTop`（不是 `adminTop`），请按实际路径调用。
- 鉴权头主流是 `token`，不要默认 `Authorization`（刷新 token 接口除外）。
- 图形验证码业务类型要与登录/发邮件 `type` 完全一致（中文值），否则会被中间件拒绝。
- 图片上传返回的是图片 `id`，取图要走 `/api/image?id=<id>`。
- 收藏夹相关接口存在隐私控制（`openCollect`），请求别人收藏夹前要处理“未开启”提示。
