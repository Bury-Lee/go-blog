## Ai 模块

- **路径**: `/chat`
- **方法**: `POST`

| --- | --- | --- | --- |
| `messages` | `Array<openai.ChatCompletionMessage>` | 否 | 对话历史消息 |
| `user_input` | `string` | 否 | 用户输入 |
| `image_ID` | `number` | 否 | 图片ID（可选）//TODO:加入图片的支持功能 |
| `model` | `string` | 否 | 模型名称（可选，默认使用全局配置） |
- **响应格式 (标准格式)**:
  ```json
  {
    "code": 0,
    "data": {},
    "msg": "成功"
  }
  ```

## Article 模块
### 创建文章
- **路径**: `/article`
- **方法**: `POST`
- **请求参数 (ArticleCreateRequest)**:
| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `title` | `string` | 是 | 文章标题，最大32字符 |
| `abstract` | `string` | 否 | 文章摘要，最大256字符 |
| `content` | `string` | 是 | 文章内容 |
| `categoryID` | `number` | 否 | 文章分类ID，关联分类表 |
| `tagList` | `Array<string>` | 否 | 标签列表，JSON序列化存储 //serializer:json要删掉?似乎要换成自己定义的taglist数据类型 |
| `cover` | `string` | 否 | 文章封面图片URL |
| `openComment` | `boolean` | 否 | 是否开启评论：true-开启 false-关闭 |
| `status` | `models.Status` | 否 | 状态,普通用户只能设置为草稿或者审核中,管理员可设置为任意值 |
- **响应格式 (标准格式)**:
  ```json
  {
    "code": 0,
    "data": {},
    "msg": "成功"
  }
  ```
