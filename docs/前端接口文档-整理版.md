# GoBlog 前端接口文档整理版

## 1. 通用约定

- 接口前缀：`/api`
- 静态资源前缀：`/web`
- AccessToken：
  - 请求头：`token: <AccessToken>`
  - 或 Query：`?token=<AccessToken>`
- 刷新 Token：
  - 推荐请求头：`Authorization: Bearer <RefreshToken>`
  - 兼容请求头：`refreshToken: <RefreshToken>`
- 统一响应：
```json
{
  "code": 200,
  "data": {},
  "message": "成功"
}
```
- 通用分页参数：
  - `page`
  - `limit`
  - `key`
  - `order`
- 通用批量删除请求体：
```json
{
  "IDList": [1, 2, 3]
}
```

## 2. 枚举速查

- 用户角色 `role`
  - `1` 管理员
  - `2` 超级会员
  - `3` 会员
  - `4` 普通用户
  - `5` 访客
  - `6` 封禁用户
- 文章状态 `status`
  - `0` 草稿
  - `1` 审核中
  - `2` 已发布
  - `3` 已下线
- 消息类型 `type`
  - `0` 评论通知
  - `1` 回复通知
  - `2` 点赞通知
  - `3` 收藏通知
  - `4` 私信
  - `5` 系统通知
  - `6` @我
- 聊天消息类型 `msgType`
  - `1` 文本
  - `2` 图片
  - `4` Markdown

## 3. 站点配置模块

### 3.1 获取 QQ 登录地址
- 接口：`GET /api/site/qq_login`
- 鉴权：否
- 请求：无
- 返回：QQ OAuth 地址字符串
- 说明：前端用于跳转 QQ 登录

### 3.2 读取站点配置
- 接口：`GET /api/site/:name`
- 鉴权：
  - `name=site` 无需管理员
  - 其他值需要管理员
- 路径参数：
  - `name`：`site | email | qq | ai`
- 返回：对应配置对象
- 说明：敏感字段已脱敏

### 3.3 更新站点配置
- 接口：`PUT /api/site/:name`
- 鉴权：管理员
- 路径参数：
  - `name`：`site | email | qq | qiNiu | ai`
- 请求体：对应配置完整 JSON
- 返回：更新结果消息
- 说明：按配置类型分别绑定

## 4. 日志模块

### 4.1 日志列表
- 接口：`GET /api/logs`
- 鉴权：管理员
- Query：
  - `page, limit, key, order`
  - `logType, level, ip, loginStatus, serviceName, userID`
- 返回：`list + count`
- 说明：支持分页与筛选

### 4.2 标记日志已读
- 接口：`GET /api/logs/:id`
- 鉴权：管理员
- 路径参数：
  - `id`：日志 ID
- 返回：结果消息
- 说明：会把对应日志标记为已读

### 4.3 删除日志
- 接口：`DELETE /api/logs`
- 鉴权：管理员
- 请求体：
```json
{
  "IDList": [1, 2, 3]
}
```
- 返回：删除结果消息
- 说明：批量删除

## 5. 图片模块

### 5.1 上传图片
- 接口：`POST /api/image`
- 鉴权：否
- 请求：
  - `multipart/form-data`
  - 字段：`file`
- 返回：`data` 为图片 ID
- 说明：取图请走图片读取接口

### 5.2 图片列表
- 接口：`GET /api/images`
- 鉴权：管理员
- Query：
  - `page, limit, key, order`
- 返回：`list + count`
- 说明：`key` 按文件名模糊搜索，返回中包含 `webPath`

### 5.3 删除图片
- 接口：`DELETE /api/image`
- 鉴权：管理员
- 请求体：
```json
{
  "IDlist": [1, 2, 3]
}
```
- 返回：删除结果消息
- 说明：这里实际字段名是 `IDlist`

### 5.4 获取图片文件
- 接口：`GET /api/image?id=<imageID>`
- 鉴权：否
- Query：
  - `id`：图片 ID
- 返回：文件流
- 说明：不是 JSON 响应

## 7. 验证码模块

### 7.1 获取图形验证码
- 接口：`GET /api/captcha`
- 鉴权：否
- Query：
  - `target`：`注册 | 重置密码 | 重置邮箱 | 用户名 | 邮箱`
- 返回：
```json
{
  "captchaID": "xxx",
  "captcha": "data:image/png;base64,..."
}
```
- 说明：提交业务请求时，业务类型要和 `target` 保持一致

## 8. 用户模块

### 8.1 发送邮箱验证码
- 接口：`POST /api/user/send_email`
- 鉴权：否
- 请求体：
```json
{
  "type": "注册",
  "email": "xx@xx.com",
  "captchaID": "xxx",
  "captchaCode": "1234"
}
```
- 返回：
  - 普通场景：`emailID`
  - 重置邮箱：`emailID + resetEmailID`
- 说明：`type` 可用值为 `注册 | 重置密码 | 重置邮箱`

### 8.2 邮箱注册
- 接口：`POST /api/user/email`
- 鉴权：否
- 请求体：
```json
{
  "emailID": "发送接口返回的emailID",
  "emailCode": "邮箱验证码",
  "password": "123456",
  "nickName": "昵称"
}
```
- 返回：
```json
{
  "AccessToken": "...",
  "RefreshToken": "..."
}
```
- 说明：建议统一使用小写 `emailID/emailCode`

### 8.3 登录
- 接口：`POST /api/user/login`
- 鉴权：否
- 请求体：
```json
{
  "type": "用户名",
  "val": "用户名或邮箱",
  "pwd": "密码",
  "captchaID": "xxx",
  "captchaCode": "1234"
}
```
- 返回：
```json
{
  "AccessToken": "...",
  "RefreshToken": "..."
}
```
- 说明：
  - `type` 只允许 `用户名 | 邮箱`
  - 开启图形验证码时，`target` 必须与 `type` 完全一致

### 8.4 获取当前登录用户详情
- 接口：`GET /api/user/detail`
- 鉴权：是
- 请求：无
- 返回：当前用户详情对象
- 说明：包含基础信息与用户配置字段

### 8.5 获取指定用户基础信息
- 接口：`GET /api/user/info/:id`
- 鉴权：否
- 路径参数：
  - `id`：用户 ID
- 返回：用户基础信息
- 说明：用于用户主页展示

### 8.6 登录日志列表
- 接口：`GET /api/user/loginlog`
- 鉴权：是
- Query：
  - `type`：`user | admin`
  - `userId, ip, addr, startTime, endTime, page, limit, key, order`
- 返回：`list + count`
- 说明：`type=user` 时只查自己

### 8.7 重置邮箱
- 接口：`PUT /api/user/resetEmail`
- 鉴权：是
- 请求体：
```json
{
  "ResetEmailID": "原邮箱ID",
  "ResetEmailCode": "原邮箱验证码",
  "emailID": "新邮箱ID",
  "emailCode": "新邮箱验证码"
}
```
- 返回：重置结果消息
- 说明：需先验证原邮箱，再验证新邮箱

### 8.8 更新个人资料
- 接口：`PUT /api/user/update`
- 鉴权：是
- 请求体：按需传任意字段
```json
{
  "avatar": "...",
  "abstract": "...",
  "likeTags": ["Go"],
  "nickName": "...",
  "Age": 18,
  "contactInfo": {"github":"..."},
  "openCollect": true,
  "openFollow": true,
  "openFans": true,
  "homeStyleID": 1
}
```
- 返回：更新结果消息
- 说明：未传字段不会更新

### 8.9 管理员更新用户
- 接口：`PUT /api/user/admin/update`
- 鉴权：管理员
- 请求体：
```json
{
  "userID": 1,
  "username": "new_name",
  "nickname": "new_nick",
  "avatar": "https://...",
  "abstract": "...",
  "role": 4
}
```
- 返回：更新结果消息
- 说明：可按需更新用户资料与角色

### 8.10 刷新 AccessToken
- 接口：`POST /api/user/token`
- 鉴权：否
- 请求头：
  - 推荐：`Authorization: Bearer <RefreshToken>`
  - 兼容：`refreshToken: <RefreshToken>`
- 返回：新的 AccessToken 字符串
- 说明：仅刷新 AccessToken

### 8.11 退出登录
- 接口：`DELETE /api/user/logout`
- 鉴权：是
- 请求头：
  - `token: <AccessToken>`
  - `refreshToken: <RefreshToken>`
- 返回：退出结果消息
- 说明：会将 AccessToken 和 RefreshToken 拉黑

## 9. 文章模块

### 9.1 创建文章
- 接口：`POST /api/article`
- 鉴权：是
- 请求体：
```json
{
  "title": "标题",
  "abstract": "摘要",
  "content": "markdown/html正文",
  "categoryID": 1,
  "tagList": ["Go", "Gin"],
  "cover": "https://...",
  "openComment": true,
  "status": 0
}
```
- 返回：创建结果消息
- 说明：文章状态由 `status` 控制

### 9.2 更新文章
- 接口：`PUT /api/article`
- 鉴权：是
- 请求体：
```json
{
  "id": 1,
  "title": "新标题",
  "abstract": "新摘要",
  "content": "新正文",
  "categoryID": 2,
  "tagList": ["A", "B"],
  "cover": "https://...",
  "openComment": true
}
```
- 返回：更新结果消息
- 说明：更新本人文章

### 9.3 文章列表
- 接口：`GET /api/article`
- 鉴权：按 `type` 决定
- Query：
  - `type`：`other | self | admin`
  - `userID`：`other` 时必填
  - `categoryID`
  - `status`
  - `page, limit, key, order`
- 返回：`ArticleListResponse` 列表
- 说明：
  - `type=other` 时，后端强制查询已发布文章
  - `type=self` 时，需登录，按传入 `status` 查询自己的文章
  - `type=admin` 时，需管理员，按传入 `status` 查询

### 9.4 搜索文章
- 接口：`GET /api/article/search`
- 鉴权：否
- Query：
  - `key`
  - `tag`
  - `type`
  - `page, limit`
- 返回：`list + count`
- 说明：基于 ES 搜索

### 9.5 文章详情
- 接口：`GET /api/article/:id`
- 鉴权：可选
- 路径参数：
  - `id`：文章 ID
- 返回：文章详情
- 说明：登录状态会影响可见范围

### 9.6 置顶文章
- 接口：`POST /api/article/top/:id`
- 鉴权：是
- 请求体：
```json
{
  "articleID": 1,
  "topType": ""
}
```
- 返回：结果消息
- 说明：虽然路径里有 `:id`，实际逻辑读取的是 Body 中的 `articleID`

### 9.7 取消置顶
- 接口：`DELETE /api/article/top`
- 鉴权：是
- 请求体：
```json
{
  "articleID": 1
}
```
- 返回：结果消息
- 说明：取消当前用户自己的置顶

### 9.8 管理员取消他人置顶
- 接口：`DELETE /api/article/admingTop`
- 鉴权：管理员
- 请求体：
```json
{
  "userID": 2,
  "articleID": 1
}
```
- 返回：结果消息
- 说明：路径拼写是 `admingTop`

### 9.9 审核文章列表
- 接口：`GET /api/article/review`
- 鉴权：管理员
- Query：
  - `userID`
  - `page, limit, key, order`
- 返回：`list + count`
- 说明：当前实际固定查询 `status=0` 的草稿文章

### 9.10 审核文章
- 接口：`POST /api/article/review/:id`
- 鉴权：管理员
- 请求体：
```json
{
  "articleID": 1,
  "status": 2,
  "msg": "备注"
}
```
- 返回：审核结果消息
- 说明：前端建议仅使用 `status=2` 或 `status=0`

### 9.11 浏览记录打点
- 接口：`POST /api/article/look`
- 鉴权：是
- 请求体：
```json
{
  "articleID": 1,
  "timeSecond": 35
}
```
- 返回：结果消息
- 说明：未登录时返回“未登录”，但仍是正常响应结构

### 9.12 点赞文章
- 接口：`POST /api/article/digg/:id`
- 鉴权：是
- 路径参数：
  - `id`：文章 ID

- 说明：再次调用会取消点赞

### 9.13 用户删除自己文章
